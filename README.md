# linear and binary research
In software development, two popular techniques for locating an item in a collection, like a list or array, are linear search and binary search. From the beginning of the list until the desired item is located or the list's end is reached,
each element is checked one at a time in a linear search. Although it works on both sorted and unsorted data and is easy to grasp, it can be slow for large datasets because, in the worst scenario, every element needs to be examined.While 
binary search, on the other hand, is much faster but has an important requirement: the data must be sorted. Instead of checking elements one at a time, binary search repeatedly
divides the list in half. It compares the target value with the middle element and then continues searching in either the left or right half, depending on the comparison.
Because it eliminates half of the remaining elements at each step, binary search is far more efficient for large datasets.
